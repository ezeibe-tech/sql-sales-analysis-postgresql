# 📊 SQL Sales Analysis Project (PostgreSQL)

## 📌 Project Overview

This project demonstrates end-to-end SQL data analysis using PostgreSQL on a real-world sales dataset. The analysis focuses on transforming raw transactional sales data into meaningful business insights through data exploration, filtering, aggregation, and advanced SQL techniques.

The project was completed as part of the **AnalystLab Africa Data Analytics Internship – Week 3 (SQL & Data Querying)**.

---

## 🎯 Project Objectives

- Analyze business sales performance using SQL
- Extract meaningful insights from transactional sales data
- Demonstrate core and advanced SQL querying techniques
- Solve real-world business problems through data analysis

## 📂 Dataset

**Dataset:** Sample Sales Data (AdventureWorks Lite Equivalent)

The dataset contains transactional sales records with information such as:

- Order Number
- Order Date
- Customer Name
- Product Line
- Product Code
- Quantity Ordered
- Sales Amount
- Country
- City
- Territory
- Deal Size
- Order Status

## 📈 Business Questions Solved

- What is the company's total revenue?
- Which product lines generate the highest sales?
- Which countries contribute the most revenue?
- Which cities generate the highest sales?
- Who are the top spending customers?
- What are the monthly and yearly revenue trends?
- Which deal sizes generate the highest revenue?
- Which sales transactions exceed the average sales value?

---

## 💡 Key Insights

- Identified the highest-performing product lines based on total revenue.
- Ranked customers according to their total spending.
- Compared sales performance across different countries and cities.
- Evaluated monthly and yearly revenue trends to understand business performance over time.
- Applied SQL window functions to rank sales performance and support business decision-making.
